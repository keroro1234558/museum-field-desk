# 展後分析台：下一版使用與表達架構

## 核心判斷

目前版本擅長「把表單填完並產出一篇文字」，但使用者真正需要的是：

1. 看完展覽後，先把還沒成形的感受留住。
2. 從零碎感受中找到自己最想說的一件事。
3. 讓工具協助組織，而不是替使用者決定觀點與語氣。
4. 每次使用後留下可回看的觀察，逐漸建立策展判斷力。

下一版應從「表單生成器」轉為「個人觀察編輯台」。

## 現況阻力

### 使用不方便

- 手機上一次展開三個階段與 14 個進度項目，容易讓人覺得必須全部完成。
- 只有 15、30 分鐘模式，缺少剛離開展場時最需要的 3–5 分鐘快速記錄。
- 所有輸出都在頁面後段，使用者填寫過程看不到內容如何逐步成形。
- 每次只能保留一份草稿，開始新展覽前必須清空，無法累積與回看。

### 不利表達自己的想法

- 一開始便要求選擇「觀看路徑、觀眾角色、策展切入點」，分析術語出現得太早。
- 固定選項容易讓使用者把感受塞進現成分類，而不是先說自己的話。
- 生成短文使用固定段落模板，未填寫時會加入系統預設判斷，容易稀釋個人語氣。
- 「補充碎片」目前沒有進入生成結果，使用者寫下的重要原句可能被忽略。
- 每次重新生成會直接覆蓋已修改的短文，缺少原稿與版本保護。
- 只有單一文章格式，未區分私人筆記、社群短文、文章提綱或長文素材。

## 建議流程

### 入口：今天要留下什麼

使用者開啟後只做兩個選擇：

- 時間：`3 分鐘快記`、`15 分鐘整理`、`30 分鐘深入`
- 目的：`留給自己`、`分享短文`、`文章素材`

若有未完成內容，入口直接顯示「繼續上次」。

### 第一段：先留下現場

一次只顯示一題，所有題目都可略過：

- 現在腦中最清楚的一個畫面是什麼？
- 身體在哪一刻有反應？
- 有哪句話是你真的想留下的？
- 哪裡讓你困惑、煩躁或意外？

每則輸入都保存為「原始碎片」，保留使用者原句。手機輸入應明確支援短句與語音輸入，不要求完整句子。

### 第二段：找出自己的主線

工具把原始碎片排列成四區，但不改寫：

- `我看見了`
- `我感覺到`
- `我的判斷`
- `我還想問`

使用者選一則最想繼續說的碎片，工具再提供相關策展概念。概念應放在觀察之後，用來命名已經發生的感受，而不是提前限制觀看。

主線使用簡單的三格結構：

- 證據：我在展場實際看見什麼？
- 解讀：這讓我怎麼理解展覽？
- 問題：它留下什麼尚未解決的問題？

### 第三段：選擇表達方式

使用者先選輸出形式，再生成：

- `一句話判斷`
- `可分享短文`
- `文章提綱`
- `策展學習筆記`

生成結果只使用使用者選取的原始碎片與主線。若系統補入銜接句，應清楚標示為建議文字，不能冒充使用者觀點。

生成後提供：

- 保留原稿
- 建立新版本
- 縮短
- 更像自己的原句
- 複製或分享

### 回看：建立自己的觀看資料庫

每次分析保存為一筆展覽紀錄，不再只有一份草稿：

- 展覽、日期、地點
- 原始碎片
- 選定主線
- 使用過的策展概念
- 文章版本

回看頁可顯示「最近常注意的面向」與「尚未追完的問題」，讓工具真正支持長期學習。

## 資料架構

不同類型內容必須分開保存：

```text
Visit
  基本資料與使用目的

Fragment
  使用者原始文字，不由系統改寫

Thread
  使用者選定的主線：證據、解讀、問題

Concept
  與主線相關的策展概念及學習提示

Output
  由指定 Fragment 與 Thread 生成的文字版本
```

最重要的不變條件：

- 原始碎片永不被生成文字覆蓋。
- 系統推論與使用者原句必須可區分。
- 重新生成建立新版本，不覆蓋已編輯內容。
- 使用者可略過任何問題，仍能得到有用輸出。
- 所有核心功能維持離線可用。

## 實作優先順序

### 第一階段：先降低使用阻力

1. 新增 3 分鐘快記模式。
2. 改為一次顯示一個階段，提供「略過」。
3. 把目前 14 項進度改成「已留下幾則可用素材」。
4. 讓補充碎片實際進入提煉結果。
5. 生成前選擇輸出形式。
6. 重新生成不覆蓋已修改短文。

### 第二階段：強化個人表達

1. 建立原始碎片與主線選取介面。
2. 將策展概念移到主線選定之後。
3. 加入「更像自己的原句」改寫方向。
4. 清楚區分使用者原句、系統整理與建議文字。

### 第三階段：支持長期學習

1. 建立多展覽紀錄與回看頁。
2. 顯示重複出現的策展概念與問題。
3. 支援兩次展覽觀察的比較。

## 成功判準

- 使用者離開展場後 3 分鐘內能留下至少三則原始碎片。
- 未填滿所有題目仍能產出有用內容。
- 生成文字能追溯到使用者選取的原句。
- 使用者修改過的文章不會被重新生成覆蓋。
- 使用者能在一分鐘內找到過去某次展覽的觀察與文章。
- iPhone Safari、加入主畫面與離線模式持續可用。
